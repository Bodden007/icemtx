# iCEMtx

Широковещательная рассылка данных в формате CSV через последовательный порт (RS-232/RS-485) по протоколу FLECS (Flexible Embedded Communication Standard). Приложение работает в режиме передачи без ожидания ответа.

## Назначение

Проект предназначен для эмуляции или передачи данных промышленных контроллеров в системах мониторинга и управления. Основные функции:

- Периодическая отправка FLECS-фреймов через последовательный порт
- Форматирование данных в соответствии со спецификацией протокола FLECS
- Поддержка graceful shutdown по сигналу Ctrl+C
- Конфигурируемые параметры порта и интервалы опроса

## Быстрый старт

### Требования

- Rust (stable edition 2024 или новее)
- Cargo (менеджер пакетов Rust)
- Доступ к последовательному порту (виртуальному или физическому)

### Сборка

```bash
# Сборка в режиме отладки
cargo build

# Сборка в режиме релиза
cargo build --release
```

### Запуск

```bash
# Запуск с параметрами по умолчанию (COM3, 9600 бод)
cargo run

./target/debug/icemtx
```

### Конфигурация

Параметры подключения настраиваются в коде (`src/app.rs`):
- Порт: `COM3` (по умолчанию)
- Скорость: `9600` бод (по умолчанию)
- Интервал опроса: `1` секунда (по умолчанию)

## Использование

### Основной цикл работы

1. Приложение создаёт флаг завершения (`shutdown`) для graceful shutdown
2. Устанавливает обработчик сигнала Ctrl+C
3. Создаёт экземпляр `Poller` с указанными параметрами порта
4. Запускает цикл опроса, который:
   - Создаёт FLECS-фрейм с тестовыми данными
   - Форматирует фрейм в строку CSV
   - Отправляет данные через последовательный порт
   - Ожидает следующий тик с учётом интервала

### Формат данных

FLECS-фрейм состоит из 32 полей, каждое из которых содержит:
- Числовое значение (с заданной точностью)
- Единицу измерения (если применимо)
- Выравнивание по ширине (12 символов)

Пример отформатированного поля: `"12.34 ppg      ,       "`

### Модули

#### Транспорт (`transport`)

- `SerialSession` - управление соединением через последовательный порт
- Методы: `new()`, `send()`, `reconnect()`
- Автоматическое закрытие порта при удалении

#### Сервисы (`services`)

- `Poller` - сервис периодического опроса
- Конфигурируемый интервал опроса (по умолчанию 1 секунда)
- Обработка флага завершения для graceful shutdown

#### Протокол (`protocol/flecs`)

- `FlecsField` - перечисление всех 32 полей FLECS
- `FlecsFrame` - контейнер для данных фрейма
- `FieldMeta` - метаданные полей (единицы, точность, ширина)
- `format_field()` - форматирование значений для передачи

## Разработка

### Добавление новых полей

1. Добавить вариант в перечисление `FlecsField` (`src/app/protocol/flecs/flecsfield.rs`)
2. Определить метаданные в методе `meta()` (`src/app/protocol/flecs/fieldmeta.rs`)
3. При необходимости обновить тестовые данные в `Poller::run()`

### Изменение параметров передачи

- Интервал опроса: изменить `Duration::from_secs(1)` в `Poller::new()`
- Параметры порта: изменить аргументы `Poller::new()` в `app.rs`
- Форматирование: настроить константы в `format.rs`

### Тестирование

```bash
# Запуск тестов
cargo test

# Проверка форматирования
cargo test --package icemtx --lib -- protocol::flecs::format
```

## Обработка ошибок

- Ошибки открытия порта выводятся в stderr и приводят к завершению приложения
- Ошибки отправки данных выводятся в stderr, но не прерывают цикл опроса
- Сигнал Ctrl+C корректно завершает приложение с освобождением ресурсов

## Зависимости

- `serialport = "4.9.0"` - работа с последовательными портами
- `ctrlc = "3.5.2"` - обработка сигнала Ctrl+C
